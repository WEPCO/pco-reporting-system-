# PCO Reporting System

This repository contains an evidence-based model of how Planning Center Online (PCO) exposes data through custom reports.

## Purpose

PCO does not expose a stable, unified reporting schema.  
Instead, data access is context-dependent and observable through templates.

This project aims to:

- Identify projection patterns in PCO reports
- Define contextual access schemas
- Normalize cross-app data models
- Enable consistent, reliable report creation
- Reduce ambiguity and prevent incorrect assumptions

## Approach

The system is built from:

1. Observed report templates (Liquid)
2. Rendered outputs (PDF / HTML)
3. UI variable references
4. Cross-pattern comparison

## Core Concepts

- Projection Patterns
- Contextual Access Schema
- Cross-Pattern Access Maps
- Normalized Data Model (in progress)

## Status

Early-stage model development  
Focused on PCO People reporting patterns
